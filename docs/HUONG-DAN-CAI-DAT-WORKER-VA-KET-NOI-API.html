02

Firebase

Project hiện tại là edumatrix-vn-576b1. Firebase cung cấp Auth, Firestore, Hosting và App Check.

Biến public phía trình duyệt

VITE_FIREBASE_API_KEY=<firebase-web-api-key>
VITE_FIREBASE_AUTH_DOMAIN=edumatrix-vn-576b1.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=edumatrix-vn-576b1
VITE_FIREBASE_STORAGE_BUCKET=<firebase-storage-bucket>
VITE_FIREBASE_MESSAGING_SENDER_ID=<sender-id>
VITE_FIREBASE_APP_ID=<firebase-web-app-id>
VITE_FIREBASE_MEASUREMENT_ID=<measurement-id-neu-co>
VITE_APPCHECK_SITE_KEY=<recaptcha-v3-site-key>

Firebase web config không phải khóa quản trị. Bảo vệ dữ liệu bằng Firestore Rules, phân quyền người dùng và App Check, không bằng cách giấu các biến VITE_FIREBASE_*.

npx firebase login
npx firebase use edumatrix-vn-576b1
npm run test:rules
npm run build
npx firebase deploy --only firestore:rules,firestore:indexes,hosting
03

Cloudflare Worker

Worker là ranh giới bảo mật: nhận Firebase ID token từ frontend, xác minh quyền rồi mới dùng secret để gọi Meta.

Không đặt secret trong wrangler.jsonc. Local dùng .dev.vars; production dùng Wrangler Secrets. Môi trường --env production là Worker riêng, thường có tên edumatrix-messenger-production.

Set-Location workers/messenger
npm install
Copy-Item .dev.vars.example .dev.vars
npm run dev
npm test
npm run build:prod

workers/messenger/.dev.vars

META_PAGE_ACCESS_TOKEN=<page-access-token>
META_APP_SECRET=<meta-app-secret>
META_WEBHOOK_VERIFY_TOKEN=<tu-tao-mot-chuoi-ngau-nhien>
FIREBASE_CLIENT_EMAIL=<service-account-client-email>
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"

Giữ nguyên ký tự \n trong private key. File này đã phải nằm trong .gitignore và tuyệt đối không commit.

npx wrangler login
npx wrangler secret put META_PAGE_ACCESS_TOKEN --env production
npx wrangler secret put META_APP_SECRET --env production
npx wrangler secret put META_WEBHOOK_VERIFY_TOKEN --env production
npx wrangler secret put FIREBASE_CLIENT_EMAIL --env production
npx wrangler secret put FIREBASE_PRIVATE_KEY --env production
npm run deploy:prod

Mỗi lệnh wrangler secret put có thể tạo và triển khai một version mới. Hãy chuẩn bị đủ giá trị, thêm secret theo một phiên cấu hình và chạy npm run deploy:prod cuối cùng.

04

Meta Messenger

Meta App kết nối với Facebook Page; webhook gọi vào Worker, còn frontend không bao giờ nhìn thấy Page Access Token.

Worker endpoints

POST /api/messenger/send
POST /api/messenger/post
POST /api/messenger/referral
GET/POST /webhook

Webhook fields

messages
messaging_postbacks
messaging_referrals

Luồng liên kết phụ huynh an toàn

Nhân viên tạo referral bằng { parentUid, studentId }. Worker xác minh vai trò và phạm vi học sinh, sinh nonce 128-bit dùng một lần, hết hạn sau 24 giờ. Link gửi cho phụ huynh có dạng:

https://m.me/<PAGE_USERNAME>?ref=<one-time-nonce>

Không tự ghép parentUid vào URL. Khi webhook nhận referral, Worker claim nonce theo giao dịch atomic và không ghi đè PSID đã có.

05

Google Drive & Picker

Giáo án có thể được chọn, tải lên và gọi lại trong ứng dụng. Triển khai hiện tại dùng Google Identity Services token model và scope drive.file.

Mô hình hiện tại không cần client secret hay refresh token. Access token được giữ trong bộ nhớ, có thời hạn ngắn; khi hết hạn, người dùng bấm kết nối lại. Upload multipart đang giới hạn 5 MB.

VITE_GOOGLE_CLIENT_ID=<oauth-client-id>.apps.googleusercontent.com
VITE_GOOGLE_PICKER_API_KEY=<restricted-api-key>
VITE_GOOGLE_PICKER_APP_ID=<google-cloud-project-number>
06

VietQR

Tích hợp hiện tại dùng API danh sách ngân hàng và Quick Link ảnh QR. Không cần API key, nhưng không tự xác nhận tiền đã vào tài khoản.

Danh sách ngân hàng

api.vietqr.io/v2/banks

Timeout 12 giây; ứng dụng chỉ giữ ngân hàng hỗ trợ chuyển khoản và BIN 6 chữ số.

Ảnh thanh toán

img.vietqr.io

Quick Link chứa BIN, số tài khoản, số tiền, nội dung và tên tài khoản. QR cần đối soát thủ công.
